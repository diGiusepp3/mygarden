# MyGarden work rules

Dit bestand is de vaste gids voor Codex in dit project.

Lees dit eerst en volg het.

## Taal

- Spreek Nederlands.
- Gebruik simpele taal.
- Wees kort en duidelijk.
- Mobile first uiteindelijk te compileren naar android app. 

## Hoofddoel

- Maak de code kleiner, netter en makkelijker te begrijpen.
- Maak de UI moderner, rustiger en mooier.
- Gebruik moderne fonts.
- Gebruik nette cards en blokken.
- Maak de dev panel beter en bruikbaarder.
- Maak planten toevoegen makkelijker.
- Maak objecten toevoegen makkelijker.
- Maak toewijzen van planten aan bedden, serres en objecten makkelijker.
- Zorg dat de serre in de moestuin selecteerbaar is bij planten.

## Werkwijze

- Volg eerst het masterplan in `MYGARDEN_MASTERPLAN.md`.
- Doe alleen expliciet wat gevraagd word en verzin niets.
- Hallucinaties zijn niet toegelaten. Strict planning volgen
- Werk op basis van de actuele code in de repo.
- Schrijf niets weg zonder dat het past in de bestaande structuur.
- Werk stap voor stap.
- Hou telkens de code schoon.
- Maak liever kleine nette stukken dan grote rommel.
- Zet nieuwe inzichten meteen in het plan als dat nodig is.
- Upload elke wijziging naar Github 
- npm run build na elke wijziging


## Code regels 

- Houd UI, logica en data zo veel mogelijk gescheiden.
- Maak kleine herbruikbare componenten.
- Maak helpers klein en duidelijk.
- Maak state logica voorspelbaar.
- Vermijd dubbele logica.
- Vermijd onnodig ingewikkelde oplossingen.
- Gebruik duidelijke namen.
- Gebruik geen verborgen magie.
- Laat bestaande ongerelateerde wijzigingen met rust.

## UI regels

- Maak de app rustiger.
- Maak de app overzichtelijker.
- Maak kaarten, knoppen en blokken consistent.
- Maak layout logisch en eenvoudig.
- Maak de dev panel strak en begrijpelijk.
- Maak forms eenvoudiger en korter.
- Maak acties duidelijk zichtbaar.

## Werkvolgorde

- Eerst audit.
- Dan opschonen.
- Dan opsplitsen.
- Dan verbeteren van UX.
- Dan visuele upgrade.
- Daarna pas verdere verfijning.

## Altijd controleren

- Werkt de wijziging nog?
- Is het eenvoudiger geworden?
- Is het mooier geworden?
- Is het logisch voor een gewone gebruiker?
- Is het niet onnodig complex?
- Is de update in lijn met het masterplan?

## Belangrijk

- Als iets niet klopt met dit bestand, volg dit bestand.
- Als iets beter moet, noteer het in het masterplan.
- Als iets groot is, hak het op.
- Als iets onduidelijk is, maak het duidelijker.
